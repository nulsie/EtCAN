# ️EtCAN(Eternal Content Addressable Network) 

**EtCAN** is a client-side, zero-knowledge, encrypted file transfer and steganography utility. It operates entirely within the browser, leveraging **Curve25519** elliptic curve cryptography and the **InterPlanetary File System (IPFS)** to facilitate secure, censorship-resistant data exchange and static site deployment.

## Core Features

  * **Zero-Knowledge Architecture:** Keys are derived locally. Passwords and private keys never leave the browser.
  * **Asymmetric Payload Encryption:** Files are encrypted via `nacl.box` (Curve25519 + XSalsa20 + Poly1305) targeting a specific recipient's public key.
  * **PRNG-Scattered Steganography:** Hide encrypted payloads within the color channels of innocent-looking PNG/JPG carrier images.
  * **Decentralized Storage:** Payloads are pinned to IPFS via the Pinata API.
  * **Multi-Gateway Probing:** Automated data retrieval across a fallback array of public IPFS gateways.
  * **Encrypted Address Book:** Locally stored, symmetrically encrypted contact list.
  * **DWeb Deployment:** Deploy entire multi-file static websites directly to IPFS.

## Architecture & Security Model

EtCAN relies on [TweetNaCl.js](https://www.google.com/search?q=https://tweetnacl.js.org/%23/) for all cryptographic operations.

### Key Derivation (PBKDF2)

Local identities are protected using **PBKDF2** to derive a symmetric key from the user's master password.

  * **Iterations:** 600,000
  * **Hash Algorithm:** SHA-256
  * **Salt:** `username_EtCAN_Secure_Salt_v1`
  * **Storage:** The asymmetric secret key (`myKeyPair.secretKey`) is symmetrically encrypted (`nacl.secretbox`) using this derived key and stored in `localStorage`.

### Payload Encryption (nacl.box)

When a file is prepared for transfer, EtCAN bundles the raw bytes with a JSON metadata header (filename, size, type). This combined buffer is encrypted using the sender's secret key and the recipient's public key.

## The Steganography Engine

Instead of sequentially replacing the Least Significant Bits (LSB) of the first few pixels (which is highly detectable via steganalysis), EtCAN uses a **Seeded Pseudo-Random Number Generator (PRNG)** to scatter the bits across the entire image.

1.  **Seed Generation:** A random 32-bit integer is generated.
2.  **Seed Embedding:** The seed itself is embedded sequentially into the first 32 valid pixel channels.
3.  **Alpha Preservation:** The alpha channel (opacity) is strictly ignored `(i + 1) % 4 !== 0` to prevent visual artifacts or transparency corruption.
4.  **Payload Scattering:** The remaining bits of the encrypted payload are scattered deterministically across the image based on the sequence generated by the PRNG seed.

## Installation & Setup

Because EtCAN is a pure static client application, there is no build step or server required.

### 1\. Clone the Repository

```bash
git clone https://github.com/yourusername/EtCAN.git
cd EtCAN
```

### 2\. Obtain a Pinata JWT

EtCAN requires a Pinata API JWT to pin files to IPFS.

1.  Create a free account at [Pinata.cloud](https://www.google.com/search?q=https://app.pinata.cloud/).
2.  Navigate to **API Keys** -\> **New Key**.
3.  Grant `pinFileToIPFS` permissions.
4.  Copy the **JWT**.

### 3\. Run the App

Simply open `index.html` in any modern web browser.

## usage Workflow

### Step 1: System Configuration

  * Paste your Pinata JWT into the **System Configuration** panel and click **Lock Configuration**. This securely encrypts the token in your local storage.

### Step 2: Identity Generation

  * Click **Generate New Keypair**.
  * Enter a Username and Auth Pass, then click **Save ID**. This derives your PBKDF2 key and securely stores your identity.

### Step 3: Outbound Transfer (Dropping a file)

  * Obtain your recipient's Public Key (PUB) and paste it into the **Recipient PUB Key** field.
  * Select your payload file.
  * *(Optional)* Toggle **Enable Steganography** and select a carrier image.
  * Click **Transmit File**. EtCAN will output an IPFS CID (Content Identifier).

### Step 4: Data Downlink (Retrieving a file)

  * Enter the Sender's PUB Key and the IPFS CID.
  * Click **Retrieve Data**. The system will probe gateways, fetch the file, extract the stego (if present), decrypt the payload, and prompt a local download.

## Code Highlights

### Intelligent Gateway Probing

To combat IPFS gateway latency, EtCAN races multiple gateways and aborts stalled requests.

```javascript
const ipfsGateways = [
    "https://ipfs.io/ipfs/",
    "https://cloudflare-ipfs.com/ipfs/",
    "https://dweb.link/ipfs/",
    "https://gateway.pinata.cloud/ipfs/"
];

// ... inside pickupFile()
for (const gateway of ipfsGateways) {
    try {
        log(`Probing: ${gateway}${cid.substring(0,8)}...`);
        // 5-second timeout signal
        const res = await fetch(`${gateway}${cid}`, { signal: AbortSignal.timeout(5000) }); 
        
        if (res.ok) {
            arrayBuffer = await res.arrayBuffer();
            successGateway = gateway;
            break; // Stop probing once a successful stream is found
        }
    } catch (e) {
        console.warn(`Gateway ${gateway} failed or timed out.`);
    }
}
```

### Deterministic Bit Scattering (PRNG)

This is the core of the steganography engine, creating an unpredictable, scattered array of indices to hide data within image pixels.

```javascript
// Mulberry32 PRNG
function createPRNG(seed) {
    let a = seed;
    return function() {
        let t = a += 0x6D2B79F5;
        t = Math.imul(t ^ t >>> 15, t | 1);
        t ^= t + Math.imul(t ^ t >>> 7, t | 61);
        return ((t ^ t >>> 14) >>> 0) / 4294967296;
    };
}

// Scatters the usable color channels based on the seed
function getScatteredIndices(seed, validChannels, neededBits) {
    const random = createPRNG(seed);
    for (let i = 0; i < neededBits; i++) {
        const j = i + Math.floor(random() * (validChannels.length - i));
        const temp = validChannels[i];
        validChannels[i] = validChannels[j];
        validChannels[j] = temp;
    }
    return validChannels.subarray(0, neededBits);
}
```
